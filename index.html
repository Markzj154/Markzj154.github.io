Django快速入门

使用虚拟环境的目的

避开多个项目之间python库的冲突
完整便捷导出python库的列表

具体步骤

安装virtualenv：pip install virtualenv
创建虚拟环境：virtualenv 虚拟环境的名称
启动虚拟环境：Scripts\activate
退出虚拟环境：deactivate

安装Django

启动虚拟环境后，通过命令 pip install django 进行安装
如果安装超时，可以通过镜像(此处是豆瓣的镜像)安装：pip install django -i http://pypi.douban.com/simple –trusted-host pypi.douban.com
如果想要指定django的版本，直接在django后面接版本即可(如：django==2.0.7)

查看Django版本

python -m django –version
django-admin –version

创建项目(blog)

django-admin startproject blog
执行上面命令后，会生成相应的文件夹和文件，他们的作用如下：
manage.py：一个让你管理Django项目的工具程序
blog/__init__.py：一个空文件，告诉python解释器这个目录应该被视为一个python包
blog/settings.py：Django项目配置文件
blog/urls.py：Django项目的URL声明，就像是你的网站的目录
blog/wsgi.py：项目运行在WSGI兼容Web服务器上的接口文件

运行项目

python manage.py runserver
执行上面命令后，通过http://127.0.0.1:8000就可以访问服务了

创建应用(user)

python manage.py startapp user
执行上面命令后，会生成user目录，其目录结构以及作用如下：
__init__.py：一个空文件，告诉python解释器这个目录应该被视为一个python包
admin.py：可以用来注册模型，用于在Django的管理界面管理模型
apps.py：当前应用的配置文件
migrations：存放和模型相关的数据迁移信息
migrations/__init__.py：一个空文件，告诉python解释器这个目录应该被视为一个python包
models.py：存放应用的数据模型，即实体类及其之间的关系
tests.py：包含测试应用各项功能的测试类和测试函数
views.py：处理请求并返回相应的函数
